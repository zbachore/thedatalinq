Fine-Grained Access Control (FGAC)

Systematic data security at the Row, Column, and Object level.

The FGAC Concept

Traditional security models often rely on "Coarse-Grained" access, where authorization at the database or schema level grants visibility to all underlying data. Fine-Grained Access Control (FGAC) implements a Least Privilege model. Sensitive data is masked and records are filtered dynamically based on the specific identity and context of the requesting user.

Column-Level Security

Non-authorized personnel may view general table attributes while PII columns, such as "Social Security Numbers," are automatically redacted or masked.

Row-Level Security

Queries against a global dataset are restricted at the engine level to return only the rows relevant to the user's regional or departmental authorization.

Databricks Implementation: Unity Catalog

In modern Databricks environments, FGAC is centrally managed via Unity Catalog. Effective implementation requires adherence to several architectural best practices:

  • Identity Federation: Linking the Databricks Account to an Identity Provider (such as Entra ID) ensures that users and groups are consistently synchronized and audited.
  • Dynamic Views & Functions: Logic is applied via SQL functions like is_account_group_member() to create views that filter or mask data based on runtime group membership.
  • Service Principal Utilization: Automated production pipelines are executed via Service Principals to maintain a clear audit distinction between human users and system processes.
  • Storage Isolation: Direct access to the underlying Storage Account (ADLS Gen2) is restricted. Data access is routed exclusively through Unity Catalog External Locations to enforce governance and prevent bypass.

Conclusion

FGAC is a foundational component of Regulatory Compliance (GDPR, HIPAA). It allows for the consolidation of data into a single source of truth while ensuring that sensitive information remains restricted to authorized personnel, significantly reducing the organization's risk surface and ensuring data integrity.
