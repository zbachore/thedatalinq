Fine-Grained Access Control (FGAC)

Securing data at the Row, Column, and Object level.

The FGAC Philosophy

Traditional security relies on "Coarse-Grained" access (e.g., if you have access to the Database, you see everything in it). Fine-Grained Access Control (FGAC) shifts this to a Least Privilege model. We mask sensitive data and filter records based on the user's specific identity and context.

Column-Level Security

An HR analyst can see "Total Compensation," but the "Social Security Number" column is automatically masked with asterisks.

Row-Level Security

A regional manager in Europe can query the global "Sales" table, but the system only returns records where Region == 'EMEA'.

Databricks Implementation: Unity Catalog

In modern Databricks architectures, FGAC is managed via Unity Catalog. To implement this effectively, we adhere to the following requirements and best practices:

  • Identity Federation: Ensure the Databricks Account is linked to Entra ID (Azure AD) so users and groups are synced correctly.
  • Dynamic Views: Use SQL functions like is_account_group_member() to create views that filter data based on the current user's group membership.
  • Service Principal Isolation: Automated pipelines should use Service Principals, not user identities, to ensure logs clearly distinguish between human and machine actions.
  • External Locations: Never grant direct access to the Storage Account (ADLS Gen2) to end users. Always route access through Unity Catalog External Locations.
⚠️ ARCHITECT'S GOTCHA: Admin Role Persistence

Never grant the Workspace Admin role to a Group. In Databricks, permissions granted at the workspace level behave differently than data permissions. If you grant a group Workspace Admin rights, the users in that group are assigned the role individually. Even if you later remove the group from the workspace, the individual users will retain their Admin status. Always assign Admin roles to specific, audited individuals or manage them strictly via Service Principals.

Why This Matters

FGAC is the foundation of Compliance (GDPR, HIPAA). It allows us to consolidate data into a single source of truth while guaranteeing that sensitive information remains visible only to authorized personnel, reducing the risk of internal data breaches.
