Medallion Architecture

Multi-hop data processing for the modern Enterprise Lakehouse.

The Multi-Hop Strategy

The Medallion architecture is a data design pattern used to logically organize data in a Lakehouse. The goal is to incrementally improve the structure and quality of data as it flows through each layer of the lake.

Bronze (Raw)

The landing zone for raw source data. No transformations are applied, maintaining full fidelity for audit trails.

Format: Parquet/JSON
Retention: Permanent

Silver (Cleared)

Data is cleaned, filtered, and joined. This layer provides an "Enterprise View" of business entities.

Format: Delta Lake
Quality: Validated

Gold (Curated)

Aggregated data ready for consumption by BI tools and Data Science models. Optimized for read performance.

Format: Delta Table
Type: Star Schema

Engineering Excellence

  • ACID Transactions: Utilizing Delta Lake to ensure data integrity during concurrent read/write operations.
  • Schema Evolution: Gracefully handling changes in source data without breaking downstream production pipelines.
  • Data Lineage: Integrated with Unity Catalog to track data flow from the raw Bronze ingestion to the final Gold KPI.

Configuration-Driven Ingestion

Instead of writing custom code for every table, The Data Linq uses a metadata-driven approach. By defining source-to-target mappings in JSON/YAML, we can deploy hundreds of pipelines using a single Spark template.
