Unity Catalog & Governance

Centralized access control and automated data lineage for Azure Databricks.

What is Data Governance?

Data governance at The Data Linq isn't just about restricting access; it's about enablement. We ensure that the right people have access to the right data at the right time, while maintaining a clear audit trail of how that data was produced.

Source System → Bronze Container → Silver Table → Gold Aggregate → Power BI
(End-to-End Lineage Tracked by Unity Catalog)

How We Govern Data

  • Attribute-Based Access Control (ABAC): Dynamically granting access based on user tags and data sensitivity.
  • Row & Column Level Security: Masking PII data automatically so analysts can see trends without seeing private identifiers.
  • Discovery & Search: A centralized UI where business users can find data assets across the entire enterprise.

Provisioning Workflow

Our standard blueprint for initializing a secure Unity Catalog environment:

1
Initialize Workspace: Provision the Azure Databricks Workspace via Terraform.
2
Access Connector: Provision an Azure Databricks Access Connector (Managed Identity).
3
IAM Handshake: Grant the Access Connector Storage Blob Data Contributor on the ADLS Gen2 account.
4
Storage Credential: Create a Unity Catalog Storage Credential tied to the Access Connector.
5
External Location: Define External Locations mapping the Credential to specific Storage Containers.
6
Catalog Hierarchy: Create the Catalog pointing to the External Location, then initialize Schemas, Volumes, and Tables.

The Value of Lineage

By utilizing Unity Catalog, we provide Impact Analysis. Before changing a column in a Silver table, you can see exactly which Gold tables and Power BI reports will be affected. This reduces downtime and increases stakeholder confidence.
